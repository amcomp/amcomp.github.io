Configuring Chrome Bookmarks in Intune

     Intune is a great tool for managing Chrome. Creating permanent bookmarks for your users can help them find company web resources much quicker. Most links can be deployed with Intune and used for Google Chrome Bookmarks In this article we will look at how to configure the policy. Requirements are a Subscription that inludes Intune, and a text editor (I prefer VSCode)

     In Endpoint Manager, Navigate to Devices > Configuration Profiles, Then Select Create > New policy

  • Navigate to Devices > Configuration Profiles
  • Select new policy
  • Choose Templates
  • Choose Administrative Templates
  • Create a name for the new policy
  • Searched Managed Bookmarks
  • When you open the policy configuration, there is a description on how the profile needs to be set up.

Copy this section, and paste it into your text editor of choice

Example value: [ { "toplevel_name": "My managed bookmarks folder" }, { "name": "Google", "url": "google.com" }, { "name": "Youtube", "url": "youtube.com" }, { "name": "Chrome links", "children": [ { "name": "Chromium", "url": "chromium.org" }, { "name": "Chromium Developers", "url": "dev.chromium.org" } ] } ]

Top Level Name is what the Folder will be named that contains all the Bookmarks you add. There is also a section of childlinks, which are a folder within the folder. You can add or subtract the bookmarks are you need, Each new bookmark needs: { "name": "Google", "url": "google.com" },

  • Once the text is configured, Go back to the “Managed Bookmarks” policy page, select “Enabled” and Paste the text in the input box below, Finally Push Ok.
  • Next Assign the Scope Tags and Assignments your environment requires, and save the policy.
  • Once devices Synchronize with the intune servers, users will be able to see the new bookmarks you added.
